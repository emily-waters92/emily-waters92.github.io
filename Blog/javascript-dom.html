An analogy to describe JavaScript and its relationship to HTML and CSS
If HTML, CSS and JavaScript were a car, HTML would be the frame or structure that makes up the car. CSS would be the paint used to make the car look nice and JavaScript would be the engine to make the car go. When all three are combined you have a car that is structurally sound, looks good and takes you where you want to go!

Explain control flow and loops using an example process from everyday life
Control flow is the order in which a computer reads code. It is normally run in order from the first line until the last line, unless something changes the control flow eg. loops. Loops are used to repeat a chunk of code a set number of times or until a condition is met. Loops are a great way to reduce your code and not have to repeat it, therefore boosting productivity in your work and efficiency in your code. An example of control flow in everyday life is my morning routine:
  • Wake up
  • Have a shower
  • Get changed
  • Have breakfast
  • Brush teeth
  • Pack Belle’s bag
  • Get in the car
  • Drive to school
An example of something (loop) that would change my morning routine (control flow) is if I were to pop a tyre. I would then have to go through the steps (loop) of changing the tyre (multiple times if I couldn’t get it right - loop) until the tyre is fixed (the condition is met).

Explain the difference between accessing data from arrays and objects
Arrays and objects are two ways of collecting data into a group. Arrays keep track of the order of the data and are useful if you want to manipulate data as a batch whilst objects are grouped properties of a single item that each have their own type.

Explain what functions are and why they are helpful
Functions in JavaScript are pieces of code that perform tasks. They are helpful because once you have created a function, they can be used again and again and you don’t need to retype that piece of code. In doing so, this not only condenses the code, but saves the programmer time.
